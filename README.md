# Brian's Landscaping — Website

A fast, SEO-optimized marketing website for **Brian's Landscaping**, a family-run
landscaper & tree service in **Bethlehem, PA** serving the Lehigh Valley.

Built with **Next.js (App Router) + TypeScript + Tailwind CSS**. Server-rendered
for SEO, mobile-first, and Core Web Vitals–friendly (no web-font network cost,
small JS payload, lazy-loaded images).

## Features

- **SEO**: per-page unique `<title>` + meta description, single H1 per page,
  canonical URLs, Open Graph/Twitter cards, clean location-keyword slugs
  (e.g. `/tree-removal-bethlehem-pa`), auto XML sitemap + robots.txt.
- **Structured data (JSON-LD)**: `LandscapingBusiness`/`LocalBusiness` with NAP,
  geo, `openingHoursSpecification`, service area, and `aggregateRating` (4.4 / 19);
  `Service` per offering; `BreadcrumbList`; `FAQPage`; `ImageGallery` + `ImageObject`;
  `Review`.
- **Dedicated service pages** — one per service, each targeting its own keyword.
- **Project gallery** — filterable by category, **before/after comparison sliders**,
  lightbox, lazy-loaded images with keyword-rich filenames/alt text.
- **Lead capture**:
  - Multi-step **Free Quote** request (services → details + photos → address →
    contact → confirm).
  - **Book an Estimate** scheduler (date + time window request).
  - **Contact** form with embedded Google Map.
  - All with **honeypot + timing** spam protection (optional reCAPTCHA v3).
- **CTAs everywhere**: sticky header, hero, every service page, footer, and a
  mobile sticky call bar — plus click-to-call throughout.

## Getting started

```bash
npm install
npm run dev        # http://localhost:3000
```

Production:

```bash
npm run build
npm run start
```

Regenerate placeholder gallery images / OG card:

```bash
npm run gen:images
```

## Configuration

Copy `.env.example` to `.env.local` and fill in what you need. **Everything is
optional for local dev** — forms validate and log to the server console even
with no email provider configured.

| Variable | Purpose |
| --- | --- |
| `NEXT_PUBLIC_SITE_URL` | Canonical site URL (sitemap, OG, JSON-LD). Set this for production. |
| `RESEND_API_KEY`, `EMAIL_FROM`, `LEAD_NOTIFY_EMAIL` | Email lead notifications + customer confirmations via [Resend](https://resend.com). |
| `TWILIO_*` | Optional SMS lead alerts to Brian. |
| `NEXT_PUBLIC_RECAPTCHA_SITE_KEY`, `RECAPTCHA_SECRET_KEY` | Optional reCAPTCHA v3 (honeypot + timing work without it). |
| `NEXT_PUBLIC_GOOGLE_MAPS_EMBED_KEY` | Optional — higher-quality map embed (a keyless embed is used otherwise). |

## Where to edit content

All business content lives in `src/lib/` as a single source of truth:

- **`business.ts`** — Name, Address, Phone (**NAP**), hours, geo, rating, service
  area. Keep NAP identical to the Google Business Profile.
- **`services.ts`** — the 8 services (copy, SEO titles/descriptions, keywords).
- **`gallery.ts`** — project items + image paths/alt text.
- **`reviews.ts`** — testimonials. **`faqs.ts`** — FAQ content.
- **`schema.ts`** — JSON-LD generators (rarely needs editing).

## Replacing placeholder photos

The gallery and OG card use tasteful **SVG placeholders** generated by
`scripts/generate-placeholders.mjs`. To use real photos:

1. Drop compressed `.jpg`/`.webp` files in `public/images/gallery/`, **keeping the
   descriptive filenames** (e.g. `tree-removal-bethlehem-pa-before.jpg`) for image SEO.
2. Update the `src` paths (and `width`/`height`) in `src/lib/gallery.ts`.
3. Replace `public/og-image.svg` with a 1200×630 `og-image.png` (and update the
   reference in `src/app/layout.tsx` + `src/lib/schema.ts`) for best social sharing.

## Before launch — checklist

- [ ] Set `NEXT_PUBLIC_SITE_URL` to the real domain.
- [ ] Confirm **business hours** in `business.ts` (currently placeholders).
- [ ] Verify the **geo coordinates** in `business.ts` against the Google pin.
- [ ] Ensure footer **NAP** matches the Google Business Profile exactly.
- [ ] Configure `RESEND_API_KEY` + `LEAD_NOTIFY_EMAIL` so leads reach Brian.
- [ ] Add real **project photos** and any additional reviews.
- [ ] Submit `sitemap.xml` in Google Search Console.

## Project structure

```
src/
  app/                 # routes (App Router)
    [serviceSlug]/     # dynamic, SSG service pages (8)
    api/               # quote / book / contact handlers
    (pages…)           # home, services, gallery, reviews, about, faq, contact, quote, book, thank-you
    sitemap.ts robots.ts manifest.ts icon.svg
  components/          # Header, Footer, Hero, Gallery, BeforeAfterSlider, forms/, …
  lib/                 # business, services, gallery, reviews, faqs, schema, leads (server)
public/images/gallery/ # placeholder project photos
scripts/               # placeholder image generator
```

## Deployment

Deploys cleanly to any Node host that supports Next.js (Vercel recommended —
zero-config). Set the environment variables above in the host's dashboard.
